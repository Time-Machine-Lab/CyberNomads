## 3. 任务上下文前提

---

required: true
description: 提前声明执行该任务所需的 Skill、工具、知识和数据文件。这些内容必须来自当前引流工作文件夹上下文，并明确相对位置。
format_instruction: 按“Skill / Tools / Knowledge / Data”分组列出；资源路径必须是相对路径。没有某类资源时写“无”。
example: |

- Skill:
  - `./skills/bilibili-web-api/SKILL.md`：用于读取 B 站平台能力与调用约束。
- Tools:
  - `./tools/filter-videos.js`：用于清洗候选视频结果。
- Knowledge:
  - `./knowledge/comment-strategy.md`：用于约束评论语气与边界。
- Data:
  - `./data/search-candidate-videos.json`：用于保存候选视频清单。

## 4. 任务流程

---

required: true
description: 按顺序详细说明达成任务目标所需的执行步骤。每一步都应尽量可操作，必要时指出应使用的 Skill、工具、知识文件或数据文件。
format_instruction: 必须使用有序列表；若步骤引用资源，必须直接写出相对路径。
example: |

1. 先阅读 `./skills/bilibili-web-api/SKILL.md`，确认视频搜索能力和调用边界。
2. 使用对应能力搜索候选视频，并将原始结果写入 `./data/search-candidate-videos.json`。
3. 若需要额外清洗数据，使用 `./tools/filter-videos.js` 处理结果。

## 5. 任务产出与协作

---

required: true
description: 明确声明本任务产出的数据结构、产出方式、保存位置、保存格式，以及下游任务如何读取和消费。
format_instruction: 必须同时说明“产出数据结构 / 产出方式 / 保存位置 / 保存格式 / 下游协作方式”；产出数据结构必须使用 Markdown 表格或代码块表达，不能只用自然语言描述；路径必须为相对路径。
example: |

- 产出数据结构:
  ```json
  [
    {
      "bvid": "string",
      "title": "string",
      "url": "string",
      "authorName": "string",
      "reason": "string",
      "collectedAt": "ISO-8601 datetime string"
    }
  ]
  ```
- 产出方式: 搜索候选视频后筛选与产品目标用户相关的内容，并写入结构化 JSON。
- 保存位置: `./data/search-candidate-videos.json`
- 保存格式: UTF-8 JSON，顶层为数组。
- 下游协作方式: `comment-on-prospects` 读取该文件，并根据 `reason` 与互动数据筛选评论目标。

## 6. 任务自检

---

required: true
description: 在任务执行结束前检查任务产出是否符合“任务产出与协作”章节的要求。若该任务执行过程中确实没有内容要产出，可以明确说明无产出原因。
format_instruction: 必须使用无序列表，逐项检查产出是否存在、结构是否合规、保存位置是否正确、下游协作是否可用；无产出任务必须写明“无产出原因”。
example: |

- 产出存在性: 已生成 `./data/search-candidate-videos.json`。
- 数据结构合规性: JSON 顶层为数组，每项包含 `bvid`、`title`、`url`、`authorName`、`reason`、`collectedAt`。
- 保存位置合规性: 产出文件位于 `./data/` 下，没有写出当前引流工作目录。
- 下游协作可用性: `comment-on-prospects` 可以按约定读取该文件。
- 无产出原因: 不适用。
