# Cybernomads 引流任务文档模板

## 1. 任务标识

---

required: true
description: 说明当前任务是谁、位于哪里、默认从哪个上下文入口开始工作。任务英文名必须稳定，可直接映射为任务文档文件名。
format_instruction: 必须使用 Markdown 无序列表，路径必须使用相对路径表示。
example: |

- 任务名称: 搜索候选视频
- 任务英文名: `search-candidate-videos`
- 任务文档路径: `./search-candidate-videos.md`
- 默认上下文入口: `./skills/bilibili-web-api`

## 2. 任务目标

---

required: true
description: 用 1 到 3 句话说明任务要解决什么问题、完成后应得到什么结果，不要混入执行细节。
example: |
本任务负责周期性搜索与产品目标用户相关的 B 站候选视频，并为后续评论任务准备可筛选的视频数据。

## 3. 执行条件

---

required: true
description: 说明任务什么时候可以执行，包括 cron 条件和依赖任务。没有某项条件时要明确写无。
format_instruction: 必须使用 Markdown 列表，cron 表达式与依赖任务分行展示。
example: |

- Cron: `0 */30 * * *`
- 依赖任务:
  - 无

## 4. 输入依赖

---

required: true
description: 逐项说明执行本任务前需要哪些输入、从哪里获取、读取时应关注什么结构或字段。
format_instruction: 每个输入项都必须包含“名称 / 来源 / 说明”三个要素；来源涉及文件时必须写相对路径。
example: |

- 名称: 候选视频列表
  来源: `./data/search-candidate-videos.json`
  说明: 包含 BV 号、标题、互动数据，用于后续筛选评论目标。

---

## 5. 可用资源

---

required: true
description: 说明执行该任务时应优先使用哪些 Skill、工具和知识文件，并明确它们的相对路径与用途。
format_instruction: 按“Skill / Tools / Knowledge”分组列出；资源路径必须是相对路径。
example: |

- Skill:
  - `./skills/bilibili-web-api/SKILL.md`：用于读取 B 站平台能力与调用约束。
- Tools:
  - `./tools/filter-videos.js`：用于清洗候选视频结果。
- Knowledge:
  - `./knowledge/comment-strategy.md`：用于约束评论语气与边界。

---

[在此处列出当前任务可用资源。]

## 6. 执行步骤

---

required: true
description: 按顺序描述任务执行流程。每一步都应尽量可操作，必要时指出应使用的 Skill、工具或知识文件。
format_instruction: 必须使用有序列表；若步骤引用资源，必须直接写出相对路径。
example: |

1. 先阅读 `./skills/bilibili-web-api/SKILL.md`，确认视频搜索能力和调用边界。
2. 使用对应能力搜索候选视频，并将原始结果写入 `./data/search-candidate-videos.json`。
3. 若需要额外清洗数据，使用 `./tools/filter-videos.js` 处理结果。

## 7. 产出与协作

---

required: true
description: 说明本任务最终会产出什么、存到哪里、下游任务如何继续使用这些结果。
format_instruction: 必须同时说明“产出文件路径”和“下游协作方式”；路径必须为相对路径。
example: |

- 产出文件: `./data/search-candidate-videos.json`
- 产出说明: 候选视频清单，供评论任务筛选潜客。
- 下游协作: `comment-on-prospects` 读取该文件并根据互动数据筛选评论目标。

## 8. 执行约束

---

required: true
description: 归纳任务执行时必须遵守的边界，例如频率限制、路径规则、禁止事项、自检要求等。
format_instruction: 必须使用无序列表，至少包含一条“路径约束”。
example: |

- 所有资源路径都必须以当前引流工作目录为根，使用相对路径表达。
- 任务产出必须写入 `./data/` 目录，不得写出当前引流工作目录之外。
- 如果执行失败，必须按任务执行 Skill 的约定回写状态与原因。
