# 失败与风险

## 这是纯接口层

本 Skill 只负责执行接口调用，不负责：

- 风险节流
- 去重
- 文案审核
- 运营节奏
- 任务状态约束

如果你需要这些能力，应由上层系统单独实现。

## 常见失败

- `403 / -403`
  通常表示权限不足或需要完整登录态。
- `352 / -352`
  常见于风控拦截。
- 评论、私信发送失败
  常见原因是 Cookie 不完整、`bili_jct` 缺失或接口风控。
- 刷新失败
  常见原因是 refresh token 失效、当前 Cookie 已不再可刷新。

## 搜索模式

- `video search` 默认先匿名
- 匿名失败且提供 Cookie 时再回退到 Cookie 搜索
- 搜不到结果不等于匿名失败

## 安全建议

- 优先使用 `--cookie-file` 或 `--cookie-env`
- 不要把明文 Cookie 长期写入脚本仓库
- 不要把这个 Skill 误当成“安全发送器”
