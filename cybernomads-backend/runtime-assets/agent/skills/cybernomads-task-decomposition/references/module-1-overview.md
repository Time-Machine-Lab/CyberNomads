# 模块 1：总览与入口

本模块解决三个问题：

1. 这个 Skill 到底要做什么
2. 这个 Skill 有哪些工具可以直接使用
3. 整个任务拆分工作流的完整顺序是什么

## Skill 目标

把一个引流工作拆成一组可以被后续 subagent 独立执行的任务，并同时完成：

- 任务结构设计
- 运行时资源选择
- 工作目录上下文准备
- 任务集保存到 Cybernomads 系统
- 全流程自检

## 可用工具

| 工具 | 路径 | 作用 |
| --- | --- | --- |
| 资源复制工具 | `scripts/copy-runtime-resource.js` | 把全局 `agent/skills` 或 `agent/knowledge` 下的资源复制到当前引流工作目录 |
| 任务批量保存工具 | `scripts/batch-save-tasks.js` | 校验 `task-set.json` 并通过受控接口批量保存任务 |
| 自检工具 | `scripts/run-self-check.js` | 检查任务结构、资源复制结果、任务保存结果是否完整闭环 |

如需查看参数、命令格式和示例，读取 [available-tools.md](available-tools.md)。

## 全流程

1. 理解引流工作目录与运行时边界
2. 识别当前可用资源
3. 设计任务元数据和任务集
4. 基于任务运行原理完成任务拆分
5. 准备工作目录中的资源、知识和数据落点
6. 把任务集归档到系统
7. 运行自检并收口
