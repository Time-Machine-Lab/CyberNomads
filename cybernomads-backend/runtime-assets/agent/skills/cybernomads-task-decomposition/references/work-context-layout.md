# 引流工作上下文目录

## 目标

先理解当前引流工作目录里每个目录是做什么的，再决定任务文档、资源和数据应当放在哪里。

## 标准结构

当前引流工作上下文目录的稳定骨架如下：

```text
<trafficWorkId>/
├── skills/
├── tools/
├── knowledge/
└── data/
```

## 目录职责

- `skills/`：只放后续 subagent 在执行任务时会直接使用的 Skill 目录。
- `tools/`：只放当前引流工作专用的可执行工具脚本，当前优先使用 `js`。
- `knowledge/`：只放该引流工作下多个任务可能共享的知识、规则、经验、产品补充说明或平台补充说明。
- `data/`：只放任务执行过程中会持续读写的产出数据或中间数据。

## 任务文档规则

- 任务文档是 Agent 在任务拆分阶段创建的产物，不是后端预置占位文件。
- 任务文档默认放在引流工作根目录下。
- 文档命名优先使用稳定英文键，例如 `task-search-candidate-videos.md`，避免依赖中文标题或平台相关临时词。
- `documentRef` 必须指向引流工作目录内部的稳定文档引用。

## 路径原则

- 所有任务文档、共享知识、工具脚本和数据文件都必须留在当前引流工作目录内部。
- 任何路径都要兼容 macOS、Windows、Linux，不依赖硬编码盘符或手写分隔符风格。
- 在任务契约和说明里优先使用工作上下文内的相对引用语义，不要把绝对路径写死到任务说明中，除非当前提示明确要求你使用该绝对路径辅助定位。
