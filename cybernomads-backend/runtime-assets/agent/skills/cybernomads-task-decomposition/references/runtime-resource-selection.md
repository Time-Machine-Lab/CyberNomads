# 运行时资源识别

## 先看什么

开始拆分前，先读取当前 Cybernomads 运行时资源清单：

- `./agent/knowledge/Agent资源清单.md`

如果当前提示已经提供了 Cybernomads 根目录绝对路径，就以那个根目录为解析基准去定位这份清单。

## 你要做的判断

围绕当前产品、策略和平台目标，判断：

- 哪些全局 Skill 需要复制到当前引流工作的 `skills/`
- 哪些全局 Knowledge 需要复制到当前引流工作的 `knowledge/`
- 哪些内容不需要复制，避免把无关资源塞进上下文

## 资源选择原则

- `cybernomads-task-execution`：默认需要复制。因为后续 subagent 执行单任务时需要统一执行规则。
- 平台 Skill：按真实平台需求复制。例如当前策略要求去 B 站执行宣传，就复制 `bilibili-web-api`。
- 共享知识：只有当多个任务都会反复用到同一批知识时，才放入 `knowledge/`。
- 临时专用能力：如果只是某个任务专属的小工具或小规则，优先做成任务文档或 `tools/` 下的工作专用脚本，不要误当成全局 Skill。

## 复制边界

- 只允许从 Cybernomads 全局 `agent/skills/` 或 `agent/knowledge/` 复制到当前引流工作上下文。
- 不要把任意本地目录当成可复制资源。
- 不要把整棵 `agent/` 目录原样复制到工作上下文；只复制当前任务真正需要的那部分。

## 资源识别结果至少要回答

- 需要复制哪些 Skill
- 需要复制哪些共享知识
- 哪些任务会使用这些资源
- 这些资源将被放到工作上下文的哪个目录
