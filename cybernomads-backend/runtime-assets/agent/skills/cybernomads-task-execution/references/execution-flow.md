# 执行流程

## 范围

只执行一个已提交的任务。不要拆分、替换或重排引流工作的任务集。

## 步骤

1. 读取任务详情并确认任务 ID。
2. 确认该任务就是规划器或调用方提交的任务。
3. 加载 `instruction`、`contextRef`、`condition` 和 `inputNeeds`。
4. 加载必需的工作上下文资产和上游产出记录。
5. 执行任务说明中要求的工作。
6. 将产出数据保存到任务数据区域或其他可引用产物位置。
7. 为产出数据创建产出记录。
8. 将状态更新为 `completed` 或 `failed`。

## 输入处理

- 将 `inputNeeds` 视为必需的数据发现说明。
- 如果输入指向上游任务，查询产出记录或加载对应任务数据区域。
- 如果必需输入无法找到，按失败规则处理，不要猜测补齐。

## 输出处理

- 将具体产出数据保存在任务契约之外。
- 通过描述和数据位置记录产出。
- 产出记录保持抽象，以支持视频、评论、私信、图片、文章或其他任务特定产物。
