# 失败规则

## 何时标记失败

- 必需上下文或输入数据缺失。
- 可用工具无法完成任务目标。
- 平台或 provider 动作出现任务内无法恢复的失败。
- 任务说明互相矛盾或不适合执行。
- 产出数据无效，或无法保存到可引用产物位置。

## 失败回写

使用受控任务状态更新 API/工具：

- 将 `status` 设置为 `failed`。
- 写入简短的 `statusReason`，说明阻塞条件。
- 不要发明 `blocked`、`cancelled`、`retrying`、`paused` 等额外状态。

## 部分产出

如果失败前已经产生有价值的部分数据：

- 将其保存到任务数据区域或产物位置。
- 创建产出记录，并清楚标注这是部分产出。
- 然后将任务标记为 `failed`，并写明原因。

## 不要掩盖失败

如果必需产出没有产生，不要标记为 `completed`。不要静默跳过必需输入，也不要用假设替代输入数据。
