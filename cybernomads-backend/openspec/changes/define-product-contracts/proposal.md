## Why

Cybernomads 已经明确了产品域的业务边界，但产品模块的顶层契约仍然缺失：目前既没有产品模块的 API 文档，也没有产品模块的 SQL 文档。按照项目宪法，任何后续产品模块实现都必须先有 `docs/api/` 与 `docs/sql/` 作为单一真理，否则实现会失去稳定边界并与领域设计脱节。

现在需要先把产品模块的对外契约和存储契约固定下来，明确产品创建、更新、列表、详情与完整上下文提供的行为，以及名称不强制唯一、不支持删除、详情返回完整正文等已确认的业务决策。

## What Changes

- 新增产品模块的顶层 API 契约文档，覆盖产品创建、更新、列表查询和详情查询。
- 新增产品模块的顶层 SQL 契约文档，定义产品模块所需的最小数据结构。
- 明确产品名称不强制唯一，避免把名称误当稳定主键。
- 明确当前阶段不支持删除产品，避免过早引入引用清理、软删/硬删和级联处理复杂度。
- 明确产品详情查询返回完整 Markdown 正文，而不是只返回摘要。
- 明确产品列表返回摘要信息，详情返回完整上下文，两者职责分离。
- 明确“提供产品完整上下文”在当前阶段复用产品详情能力，而不是单独拆出新的外部接口。

## Capabilities

### New Capabilities
- `product-contracts`: 定义产品模块的 API 契约、存储契约以及对外可见的业务行为边界。

### Modified Capabilities

None.

## Impact

- Affected docs:
  - `docs/api/` 下新增产品模块接口文档
  - `docs/sql/` 下新增产品模块数据结构文档
- Affected systems:
  - 产品模块未来的 HTTP Controller 边界
  - 产品模块未来的 SQLite 持久化结构
- Affected behavior:
  - 产品列表只返回摘要
  - 产品详情返回完整正文
  - 不支持产品删除
  - 产品名称不强制唯一
