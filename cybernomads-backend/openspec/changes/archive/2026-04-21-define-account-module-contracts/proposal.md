## Why

Cybernomads 已经完成了账号池领域设计文档，账号模块的领域边界、聚合关系和关键业务约束已经基本明确。但当前后端侧仍然没有账号模块的顶层 API 文档、SQL 文档和运行时规格说明，这意味着后续任何实现都会缺少稳定契约，并且容易重新退回到按前端 mock 或页面形态反推后端的错误路径。

现在需要先把账号模块的文档真相层补齐，明确账号模块的最小数据结构、外部接口边界、运行时行为要求，以及当前生效凭证、待验证授权尝试、三维状态、逻辑删除与恢复等关键决策如何落到后端设计中。

## What Changes

- 新增账号模块的顶层 API 契约文档，覆盖账号创建、列表、详情、更新、授权尝试、授权验证、可用性检查、逻辑删除和恢复等行为。
- 新增账号模块的顶层 SQL 契约文档，定义账号模块所需的最小结构化存储边界。
- 明确账号详情只返回脱敏管理视图，不返回原始凭证数据。
- 明确当前生效凭证与待验证授权尝试分离，避免扫码授权、Token 提交等待验证流程污染当前生效凭证。
- 明确账号模块的三维状态语义以及最低可消费条件，避免后续实现中把生命周期、授权状态和可用状态混成单一字段。
- 明确账号模块当前只支持逻辑删除，不支持物理删除；同一平台账号重新接入时恢复原对象而非新建重复对象。
- 新增账号模块的运行时规格说明，定义后端在当前阶段必须支持的最小业务能力与约束。

## Capabilities

### New Capabilities
- `account-contracts`: 定义账号模块的 API 契约、存储契约以及对外可见的业务行为边界。
- `account-module-runtime`: 定义账号模块在后端运行时的行为要求，包括授权尝试、当前凭证切换、状态消费规则、逻辑删除与恢复。

### Modified Capabilities

None.

## Impact

- Affected docs:
  - `docs/api/` 下新增账号模块接口文档
  - `docs/sql/` 下新增账号模块数据结构文档
  - `docs/design/domain/Cybernomads账号池领域设计文档.md` 将作为账号模块规格说明的主要语义来源
- Affected systems:
  - 未来账号模块的 HTTP Controller 边界
  - 未来账号模块的 SQLite 元数据结构
  - 未来账号模块的凭证与授权尝试存储策略
- Affected behavior:
  - 账号列表返回摘要视图
  - 账号详情返回脱敏管理视图
  - 当前生效凭证只能通过受控解析能力消费
  - 授权尝试与当前凭证严格隔离
  - 只有满足最低消费条件的账号才进入可执行资源集合
